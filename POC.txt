K2 Protocol Lifecycle Validation — Proof of Concept Summary

Target: K2 Lending Protocol
Contest: Code4rena 2026-04 K2
Issue Type: Repayment boundary / amount-sensitive repayment failure
Primary lifecycle tested: supply → borrow → repay → withdraw

Summary:
This repository documents a reproducible repayment-boundary condition observed through the official K2/C4 protocol test harness.

The core finding is not based on standalone arithmetic simulation. The protocol-level evidence was produced through executable lifecycle testing using the real K2/C4 harness and protocol calls.

Observed behavior:

- Repeated patterned borrow/repay cycles remain coherent through pre-boundary execution.
- Debt returns to zero after successful pre-boundary repayments.
- Collateral remains stable during completed cycles.
- Health factor returns to max/infinite after successful repayments.
- At the known boundary window, the larger repayment path fails with UnderlyingTransferFailed / allowance-related behavior.
- Retry attempts for the larger repayment continue failing after the boundary.
- Smaller fragmented repayments can still reduce debt.
- Collateral withdrawal remains available under normal health-factor limits.
- Health-factor enforcement eventually blocks unsafe withdrawal.
- Failed operations observed in later validation phases do not mutate state unexpectedly.

Important limitation:
This repository does not claim direct reserve theft, protocol insolvency, full health-factor bypass, or confirmed unauthorized asset extraction.

Current validated claim:
The protocol can enter a deterministic post-boundary state where larger repayment attempts fail while active debt remains present, smaller fragmented repayments may still succeed, and collateral operations continue subject to normal health-factor enforcement.

Why this matters:
Repayment is a core safety path in a lending protocol. If a borrower or integration expects a single larger repayment to restore account health, this boundary condition may create repayment reliability risk, liquidation-defense friction, or recovery-path inconsistency.
